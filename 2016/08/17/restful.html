REST

  • 每一个 URI 代表一种资源
  • 客户端和服务端，传递这种资源的某种表现形式。
  • 客户端通过4个HTTP动词，对服务器资源进行操作，实现“表现层状态转化”。

HTTP 动词

  • GET 获取资源
  • POST 新增资源
  • PUT 更新资源
  • DELETE 删除资源

同时支持 HTTP 和 HTTPS 协议。

  • 在服务端，HTTP 函数既可以同步调用，也可以异步调用。
  • 在客户端，HTTP 函数只能异步调用，并传递一个回调函数。（即必须有回调函数）
